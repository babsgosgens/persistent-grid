Reference Manual

The Reference Manual is split up into sections corresponding to the parts that make up Persistent Grid. These parts are: Grid, Base, Modules and Helpers.

Each section has a full reference on each of its modules. Each module is organized in the same way, consisting of helpers, settings and behavior:


Helpers

Helpers is the general terminology for module mixins and module functions. They are assigned with module specific tasks. Helpers are always loaded first.

Settings

Each module contains it’s own module specific settings. Settings are loaded before behavior and can be overridden by your project settings.

Behavior

Behavior is what’s actually outputting style rules - if any. Behavior is controlled by the settings, and only behavior that will be used is compiled. Behavior is loaded last so that any changes to the settings can be applied before the styles are compiled.
