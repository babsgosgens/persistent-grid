Introduction

Persistent Grid is a configurable grid that is not restricted to any fixed number of columns. The module's base behaviour will only render the column styles as defined in the $column-widths array. Columns can be nested to any depth with consistent column spacing - no matter how deeply nested the column is. For a typical eight or twelve column grid, you will usually only need to provide a few widths and achieve more complex layouts by combining them. Examples are provided below.

Persistent Grid uses padding for column spacing. That way, you can apply backgrounds and have them stretch beyond the column's content. If you need to, you can swap padding with margin (using the provided mixins).

For nested containers, you can either use the container class which will render a column without padding, or apply an additional class of container to remove the padding.

Settings

Root

You can embed your grid layout in a 'root' container. Your grid layout will not stretch beyond the maximum width of this container. The root container can be positioned left or center, for alternate positions you can provide your own behavior.

$root-selector
The default selector for the root container is .root, but you can also use a base selector, for instance body.
$root-max-width
Sets the maximum width on your container. Your grid layout will not stretch beyond this width. Default setting is 1152px. Just because.
$root-position
Position your root container left or center of the viewport. Default setting is center.

Columns

$column-selector
The selector of your column rules. It is set to .column by default. Change it to any selector that has your fancy.
$column-spacing
The column-spacing sets the distance between adjacent column's content. By default, column spacing is set by applying padding, but can be overridden with margins. See: @mixin column-spacing. The default setting for $column-spacing is twice the project's preset line-height, but can be changed to any measure - but only in pixels.
$column-spacing-unit
This setting will likely be removed in the final version.
$column-widths
Expects an array of fractions. The grid's default behavior will render rules based on these values. The default column-widths are 1/1, 1/2, 1/3, 2/3 and 3/4 as these values are all that are necessary to create more complex 3, 8, 9, 12 and/or 16 column layouts. Omit and add column-widths at will.
$column-width-default
This sets the default width of a column. If no width-selector is applied, the column will be of this width. It set to 1/1 (full width) by default.
$column-breakpoint
This setting expects an array of breakpoints. Breakpoints are key/value pair(s) of valid media expressions. Set to false by default. When set, it will provide basic responsive behavior by
  1. stacking columns (width: 100%) - unless of course the column is persistent,
  2. halving column-spacing,
  3. halving bleed-margins.
$column-float
If you prefer floated columns over of inline-blocks, set to true.

Fall back for inline-block style columns

If you cannot control your markup, you can assign a negative right margin to close gaps between columns to make tem fit. This is a hack and only nescessary when html (or column markup) is minimized.

$negative-right-margin
Adjust the value to your typography, as it will depend on your type's em-width. Usually a value of 0.175 seems to produce accurate results. By default, this setting is set to false.

Mixins

@mixin column-behavior($reset-box-model: $reset-box-model, $column-float: $column-float)
Mixin basic column properties.
@mixin column-width($fraction: $base-column-width, $persistent: false)
Mixin a width. Widths are expressed as fractions and are converted to percentages by the mixin. If a $column-breakpoint is set, the columns will be stacked - unless it was defined as persistent.
@mixin column-spacing($fraction: 1/2, $use-margin: false)
Mixin column spacing. Uses padding by default, but can be overridden to use margins instead - helpful when assigning backgrounds or borders. If a $column-breakpoint is set, the value will be halved below the breakpoint.
@mixin column($fraction: 1/1, $persistent: false, $column-padding: 1/2, $column-margin: false)
Shorthand mixin for column, combines above mixins. Behavior can be overridden with the parameters.
@mixin root($root-max-width: $root-max-width)
Fixes content width and centers in viewport.
@mixin container-behavior()
Deprecated?
@mixin move($fraction)
Helper mixin for soure ordering.
@mixin bleed()
Mixin to reverse padding on a container. Is helpful with nested columns. If a $column-breakpoint is set, the margins will be halved below the breakpoint.

Usage

Columns can be rendered by:

  1. assigning classes at markup level,
  2. extending the rendered behavior, or
  3. using the provided mixins.

In the real world, you will likely use a mixture of all three. Persistant Grid does not enforce any particular system - it is totally up to you.

Using classes is the simplest way of assigning columns to your markup, and - as you can see in the examples below - uses the least amount of code. Using mixins is a little more code intensive, but useful when defining behavior that is not rendered by default. Extending classes can be useful in situations where you cannot alter the markup, but beware - it can produce bloated stylesheets in a blink.
