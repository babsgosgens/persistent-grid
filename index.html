Installing Persistent Grid (pre-alpha)

During the development of Persistent Grid (or at least until a first release candidate) you can install Persistent Grid as an Ad-Hoc Compass extension. Ad-Hoc extensions reside in a directory named extensions in the root of your Compass project.

Adding Persistent Grid to your project

If it doesn't exist yet, create a directory named extensions in the root of your Compass project directory. This is where the Persistent Grid code will reside.

Using git submodule

There are two ways to add Persistent Grid to your extensions. If you are using git for version control, you may want to add Persistent Grid as a submodule to your project. From the root of your repository:

git submodule add https://github.com/babsgosgens/persistentgrid.git path_to_your_compass_directory/extensions/persistentgrid/

Make sure to replace "path_to_your_compass_directory" with the actual path to your Compass directory.

Using an archive

You can also add Persistent Grid by extracting an archived copy of Persistent Grid into the extensions directory. Make sure to rename the extracted directory from "persistentgrid-master" to persistentgrid.

Checking that Persistent Grid is installed succesfully

From the root of your compass project's directory, run: compass frameworks

You should see Persistent Grid listed:

Available Frameworks & Patterns:
    ...
  * persistentgrid
    - persistentgrid/basic              - The basics: a simple but flexible grid system.
    - persistentgrid/framework          - A solid base for your projects. Includes the grid and base styles.
    - persistentgrid/framework-extended - Like framework, but includes styles for common UI patterns.

For more information on Compass Ad-Hoc extensions, please visit http://compass-style.org/help/tutorials/extensions/.
